Deployment Guide

Complete deployment instructions for Firebase Hosting, Azure App Service, and development setup

Development Environment Setup

Get FinBuddy running locally for development with this comprehensive setup guide.

Prerequisites

Required Software

  • Node.js 18+ (LTS recommended)
  • npm 9+ or yarn 1.22+
  • Git for version control
  • Firebase CLI tools
  • VS Code (recommended)

Firebase Project Setup

  • Firebase project with Firestore
  • Authentication (Google provider)
  • Storage bucket configured
  • Functions v2 enabled
  • Hosting service enabled

Step-by-Step Setup

  1. Clone Repository

    git clone https://github.com/theharithsa/budget-buddy.git
cd budget-buddy

  2. Install Dependencies

    npm install
# or
yarn install

  3. Configure Firebase

    Update src/lib/firebase.ts with your project configuration:

    const firebaseConfig = {
  apiKey: "your-api-key",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project.appspot.com",
  messagingSenderId: "123456789",
  appId: "your-app-id"
};

  4. Environment Variables

    Create .env file in project root:

    # Optional - for AI features
VITE_OPENAI_API_KEY=your-openai-api-key

# Optional - for observability
VITE_ENVIRONMENT=development
VITE_DYNATRACE_URL=your-dynatrace-url
VITE_DYNATRACE_TOKEN=your-dynatrace-token

  5. Start Development Server

    npm run dev
# Server will start on http://localhost:5000

  6. Deploy Functions (Optional)

    # Install Firebase CLI if not already installed
npm install -g firebase-tools

# Login to Firebase
firebase login

# Deploy functions for KautilyaAI Co-Pilot
firebase deploy --only functions

‚úÖ Development Ready

Your development environment is now ready! Visit http://localhost:5000 to see FinBuddy running locally.

Firebase Hosting Deployment

Deploy FinBuddy to Firebase Hosting for seamless integration with Firebase backend services.

Prerequisites

# Install Firebase CLI globally
npm install -g firebase-tools

# Login to Firebase
firebase login

# Initialize Firebase in your project (if not already done)
firebase init

Firebase Configuration

Ensure your firebase.json is configured correctly:

{
  "hosting": {
    "public": "dist",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**"
    ],
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ],
    "headers": [
      {
        "source": "**/*.@(js|css)",
        "headers": [
          {
            "key": "Cache-Control",
            "value": "max-age=31536000"
          }
        ]
      },
      {
        "source": "/manifest.json",
        "headers": [
          {
            "key": "Cache-Control", 
            "value": "max-age=86400"
          }
        ]
      }
    ]
  },
  "functions": {
    "source": "functions",
    "runtime": "nodejs18"
  }
}

Deployment Steps

  1. Build Production Bundle

    npm run build

  2. Deploy to Firebase

    # Deploy hosting only
firebase deploy --only hosting

# Deploy hosting and functions
firebase deploy --only hosting,functions

# Deploy everything
firebase deploy

  3. Verify Deployment

    Check your Firebase console for the deployed URL and verify all features work correctly.

Preview Channels

# Create a preview channel for testing
firebase hosting:channel:deploy preview-feature

# Deploy to a specific preview channel
firebase hosting:channel:deploy --expires 7d my-feature

Azure App Service Deployment

Deploy FinBuddy to Azure App Service for enterprise-grade hosting with advanced monitoring.

Azure Prerequisites

Azure Setup

  • Azure subscription active
  • Azure CLI installed
  • Resource group created
  • App Service plan configured

Application Settings

  • Node.js 18+ runtime
  • Environment variables configured
  • Custom domain (optional)
  • SSL certificate

Deployment Script

Use the provided PowerShell script scripts/deploy-to-azure.ps1:

# Azure deployment script
param(
    [Parameter(Mandatory=$true)]
    [string]$ResourceGroup,
    
    [Parameter(Mandatory=$true)]
    [string]$AppServiceName,
    
    [string]$Location = "East US"
)

Write-Host "Building application..." -ForegroundColor Green
npm run build

Write-Host "Creating deployment package..." -ForegroundColor Green
Compress-Archive -Path "dist/*" -DestinationPath "budget-buddy-deploy.zip" -Force

Write-Host "Deploying to Azure App Service..." -ForegroundColor Green
az webapp deployment source config-zip --resource-group $ResourceGroup --name $AppServiceName --src "budget-buddy-deploy.zip"

Write-Host "Deployment completed!" -ForegroundColor Green

Manual Deployment Steps

  1. Build for Production

    npm run build

  2. Create App Service

    az webapp create --resource-group myResourceGroup --plan myAppServicePlan --name budget-buddy-app --runtime "NODE|18-lts"

  3. Configure App Settings

    az webapp config appsettings set --resource-group myResourceGroup --name budget-buddy-app --settings WEBSITE_NODE_DEFAULT_VERSION="~18"

  4. Deploy Application

    az webapp deployment source config-zip --resource-group myResourceGroup --name budget-buddy-app --src dist.zip

CI/CD Pipeline Setup

Automate deployments with continuous integration and deployment pipelines.

GitHub Actions (Firebase)

# .github/workflows/firebase-deploy.yml
name: Deploy to Firebase Hosting

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
    
    - name: Install dependencies
      run: npm ci
    
    - name: Run tests
      run: npm run test:run
    
    - name: Build application
      run: npm run build
    
    - name: Deploy to Firebase
      uses: FirebaseExtended/action-hosting-deploy@v0
      with:
        repoToken: ${{ secrets.GITHUB_TOKEN }}
        firebaseServiceAccount: ${{ secrets.FIREBASE_SERVICE_ACCOUNT }}
        channelId: live
        projectId: your-firebase-project-id

Azure DevOps Pipeline

# azure-pipelines.yml
trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

variables:
  buildConfiguration: 'Release'

steps:
- task: NodeTool@0
  inputs:
    versionSpec: '18.x'
  displayName: 'Install Node.js'

- script: |
    npm ci
    npm run build
  displayName: 'npm install and build'

- task: ArchiveFiles@2
  inputs:
    rootFolderOrFile: 'dist'
    includeRootFolder: false
    archiveType: 'zip'
    archiveFile: '$(Build.ArtifactStagingDirectory)/budget-buddy-$(Build.BuildId).zip'
    replaceExistingArchive: true

- task: AzureWebApp@1
  inputs:
    azureSubscription: 'Azure-Service-Connection'
    appType: 'webApp'
    appName: 'budget-buddy-app'
    package: '$(Build.ArtifactStagingDirectory)/budget-buddy-$(Build.BuildId).zip'

Deployment Troubleshooting

Common Issues

Build Failures

  • Clear node_modules and reinstall
  • Check TypeScript errors
  • Verify environment variables
  • Update dependencies to latest versions

Firebase Deployment Issues

  • Check Firebase CLI authentication
  • Verify project ID in firebase.json
  • Ensure sufficient Firebase quotas
  • Check Firestore security rules

Runtime Errors

  • Check browser console for errors
  • Verify Firebase configuration
  • Test authentication flow
  • Check network connectivity

Performance Issues

  • Analyze bundle size with tools
  • Optimize images and assets
  • Implement code splitting
  • Use React DevTools Profiler
