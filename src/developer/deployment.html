Azure App Service Deployment

‚òÅÔ∏è Azure App Service
Recommended

Best for: Enterprise deployment with integrated monitoring and scaling capabilities

Deployment Steps

1
Create Azure App Service
# Using Azure CLI
az login
az group create --name budget-buddy-rg --location "East US"

az appservice plan create \
  --name budget-buddy-plan \
  --resource-group budget-buddy-rg \
  --sku F1 \
  --is-linux

az webapp create \
  --name budget-buddy-app \
  --resource-group budget-buddy-rg \
  --plan budget-buddy-plan \
  --runtime "NODE:18-lts"
2
Configure App Settings
# Set environment variables
az webapp config appsettings set \
  --resource-group budget-buddy-rg \
  --name budget-buddy-app \
  --settings \
    VITE_ENVIRONMENT=production \
    VITE_OPENAI_API_KEY="your-api-key" \
    SCM_DO_BUILD_DURING_DEPLOYMENT=true \
    WEBSITE_NODE_DEFAULT_VERSION="18.x"
3
Deploy via GitHub Actions

Create .github/workflows/azure-deploy.yml

name: Deploy to Azure App Service

on:
  push:
    branches: [ main ]
  workflow_dispatch:

env:
  AZURE_WEBAPP_NAME: budget-buddy-app
  AZURE_WEBAPP_PACKAGE_PATH: '.'
  NODE_VERSION: '18.x'

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Node.js
      uses: actions/setup-node@v4
      with:
        node-version: ${{ env.NODE_VERSION }}
        cache: 'npm'
    
    - name: Install dependencies
      run: npm ci
    
    - name: Build application
      run: npm run build
      env:
        VITE_ENVIRONMENT: production
        VITE_OPENAI_API_KEY: ${{ secrets.VITE_OPENAI_API_KEY }}
    
    - name: Deploy to Azure App Service
      uses: azure/webapps-deploy@v2
      with:
        app-name: ${{ env.AZURE_WEBAPP_NAME }}
        publish-profile: ${{ secrets.AZURE_WEBAPP_PUBLISH_PROFILE }}
        package: dist

Required GitHub Secrets

  • AZURE_WEBAPP_PUBLISH_PROFILE - Download from Azure Portal
  • VITE_OPENAI_API_KEY - Your OpenAI API key
4
Configure SPA Routing

Create web.config in public folder for SPA routing:

<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="React Routes" stopProcessing="true">
          <match url=".*" />
          <conditions logicalGrouping="MatchAll">
            <add input="{REQUEST_FILENAME}" matchType="IsFile" negate="true" />
            <add input="{REQUEST_FILENAME}" matchType="IsDirectory" negate="true" />
          </conditions>
          <action type="Rewrite" url="/" />
        </rule>
      </rules>
    </rewrite>
  </system.webServer>
</configuration>

‚ñ≤ Vercel
Easy Setup

Best for: Quick deployment with automatic builds and excellent DX

1
Connect Repository
  1. Visit vercel.com and sign in
  2. Click "New Project" and import your GitHub repository
  3. Vercel will auto-detect the Vite framework
  4. Configure environment variables in project settings
2
Configure Build Settings

Create vercel.json

{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "framework": "vite",
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ],
  "headers": [
    {
      "source": "/sw.js",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "no-cache, no-store, must-revalidate"
        }
      ]
    },
    {
      "source": "/manifest.json",
      "headers": [
        {
          "key": "Content-Type",
          "value": "application/manifest+json"
        }
      ]
    }
  ]
}

  • Add VITE_OPENAI_API_KEY (if needed)

Netlify Deployment

üåü Netlify
Popular Choice

Best for: Static site hosting with excellent PWA support and edge functions

1
Connect Repository
  1. Visit netlify.com and sign in
  2. Click "New site from Git" and connect GitHub
  3. Select your Budget Buddy repository
  4. Configure build settings
2
Build Configuration

Create netlify.toml

[build]
  command = "npm run build"
  publish = "dist"

[build.environment]
  NODE_VERSION = "18"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

[[headers]]
  for = "/sw.js"
  [headers.values]
    Cache-Control = "no-cache, no-store, must-revalidate"

[[headers]]
  for = "/manifest.json"
  [headers.values]
    Content-Type = "application/manifest+json"

[[headers]]
  for = "/*.js"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"

[[headers]]
  for = "/*.css"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"
3
Environment Variables

Configure environment variables in Netlify dashboard:

  • Go to Site Settings > Environment Variables
  • Add VITE_ENVIRONMENT=production
  • Add VITE_OPENAI_API_KEY (if needed)
  • Trigger new deploy

Security Rules Deployment

Deploy Firestore Rules

# Install Firebase CLI
npm install -g firebase-tools

# Login to Firebase
firebase login

# Initialize Firebase in project
firebase init firestore

# Deploy security rules
firebase deploy --only firestore:rules

# Deploy storage rules
firebase deploy --only storage

Firestore Security Rules

// firestore.rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can only access their own data
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    
    // Public categories accessible to all authenticated users
    match /publicCategories/{document} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == resource.data.createdBy;
    }
    
    // Public people accessible to all authenticated users
    match /publicPeople/{document} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == resource.data.userId;
    }
  }
}
